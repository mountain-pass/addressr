- **The external-comms gate cannot clear `gh issue create --body-file` â€” use the quoted-heredoc `--body "$(cat <<'EOF' ... EOF)"` form.** The gate hashes the body it extracts from the command text to match the reviewer's marker; for `--body-file` it extracts an EMPTY draft (gate source: "When absent (npm publish, --body-file...) DRAFT=''"), while the deny message tells you to have the reviewer wrap the full body in `<draft>` â€” the two keys can never match, so the deny loops forever. The heredoc `--body` form is extracted verbatim and matches a full-body review marker on the first retry. Hit 2026-07-19 filing agent-plugins#368 (two wasted reviewer dispatches before reading the hook source). Same marker-key-mismatch family as the P048 hash-exactness friction (parked, upstream #361).
- **Releasing needs a manual GitHub Actions approval both times.** The changesets release PR (`changeset-release/master`, "chore: release") has its CI runs gated as `action_required` â€” a human must approve them in the Actions UI before they run, so `release:watch` stalls waiting for a `build` check that never appears.
